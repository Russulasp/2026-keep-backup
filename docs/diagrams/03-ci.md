# 03-ci — CIフロー図（.github/workflows 抽出）

- 入口:
  - PR: `.github/workflows/no-profile-smoke-pr.yml`
  - push/main・手動: `.github/workflows/backup-ci.yml`
  - 手動: `.github/workflows/secrets-notify-check.yml`
- 出口:
  - smoke結果判定（`summary success=true` 必須）
  - 必要時ログartifactアップロード
  - 通知メール送信（設定時）
- 主要関数/実行コマンド:
  - `docker compose build app`
  - `docker compose run --rm app uv lock --check`
  - `docker compose run --rm app uv run --no-sync python -m keep_backup.app --mode smoke-playwright-fixture`
- テストコマンド（ローカル再現）: `uv run python -m unittest` + 各workflowと同等コマンド。
- CIコマンド: 下図ノード参照。

```mermaid
flowchart TD
    PR[PR to main] --> WF1[fixture-smoke-pr
(.github/workflows/no-profile-smoke-pr.yml)]
    PUSH[push to main / workflow_dispatch] --> WF2[backup-ci.yml]
    MANUAL[workflow_dispatch] --> WF3[secrets-notify-check.yml]

    subgraph WF1 [PR Smoke Workflow]
      A1[checkout]
      A2[docker compose build app]
      A3[docker compose run --rm app uv lock --check]
      A4[docker compose run --rm app uv run --no-sync python -m keep_backup.app --mode smoke-playwright-fixture]
      A5[stdoutから summary/error を検証]
      A6{failure?}
      A7[logs/run_*.log を artifact upload]
      A1-->A2-->A3-->A4-->A5-->A6
      A6--yes-->A7
    end

    subgraph WF2 [Main Smoke + Notify Workflow]
      B1[checkout]
      B2[docker compose build app]
      B3[docker compose run --rm app uv lock --check]
      B4[docker compose run --rm app uv run --no-sync python -m keep_backup.app --mode smoke-playwright-fixture]
      B5[stdout解析して GITHUB_OUTPUT へ保存]
      B6[always: send mail]
      B1-->B2-->B3-->B4-->B5-->B6
    end

    subgraph WF3 [Notify Secrets Check]
      C1[必須Secretsの存在/形式チェック]
      C2[SMTP認証テスト]
      C3[成功時: テスト通知メール送信]
      C1-->C2-->C3
    end
```

## 要確認メモ
- PR/main ともに Docker コンテナ導線でスモークを実行するため、実行基盤は同一化されている。
- CI は fixture smoke（profile-independent）を維持し、ログイン済みプロファイルは持ち込まない。
